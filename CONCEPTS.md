# Concepts

Shared domain vocabulary for this project — entities, named processes, and status concepts with project-specific meaning. Seeded with core domain vocabulary from the generation pipeline, then accretes as ce-compound and ce-compound-refresh process learnings; direct edits are fine. Glossary only, not a spec or catch-all.

---

## Generation Pipeline

### Slot
The time-of-day or mood descriptor passed as the CLI argument to `generate.py` (e.g., `"late night slow techno synths"`). The Slot drives the entire generation — it is the only human input per run.

### Brief
The prose output of the Producer bot — 2-4 paragraphs setting the scene, narrative, and atmosphere for a Slot. The Brief is the DJ bot's primary input and is saved to the Show's prompts directory as `set-<slot>.md`. It is not stored back into the seed prompt files.

### Show
A single generation run — one invocation of `generate.py` with a Slot and track count. Each Show produces its own timestamped output directory (`output/<slug>-<timestamp>/`) containing tracks, DJ drops, and a prompts subdirectory. Shows are self-contained and never share state.

### Track
A single generated MP3 audio file within a Show. Each Track is produced by ACE-Step from a Caption and lyrics, and is named with its sequence number and a slug of its Caption.

### Caption
The technical sound-design description field sent to ACE-Step as its generation prompt. A Caption names specific synth types, frequency ranges, drum sounds, and effects — it is explicitly not prose or poetry. The Caption is the primary lever for ACE-Step output quality; literary or metaphorical language degrades results.

### DJ Drop
A short TTS voiceover generated by Kokoro from Announcer bot text, inserted between Tracks at random intervals (every 2-4 Tracks). Saved as an MP3 alongside the Track it follows.

## Bots

### Producer
The LLM bot responsible for writing the Brief from a Slot. Driven by the seed prompt in `prompts/producer.md`. Runs once per Show.

### DJ
The LLM bot responsible for converting the Brief into structured Track data (Caption, lyrics, BPM, key, duration) for ACE-Step. Runs once per Track. Driven by the seed prompt in `prompts/dj.md`.
*Avoid: Writer, Writer bot* (renamed to DJ in this project)

### Announcer
The LLM bot responsible for generating the spoken text for DJ Drops. Driven by the seed prompt in `prompts/announcer.md`. Produces varied content types: thoughts, news, trivia, announcements, weather.

### Seed Prompt File
A static, human-authored Markdown file in `prompts/` that defines a bot's system prompt. Seed prompt files are read-only within the pipeline — the pipeline must never write generated output back to them. Generated artifacts (Briefs, set notes) go to the Show's output directory instead.

## Player Controls

### audioInputGain
A global (`window.audioInputGain`, 0-100, default 100) that scales the raw frequency analysis data before it reaches visual sketches. Controlled by the **sensitivity** knob in the deck. Reduces visual reactivity to audio without affecting playback volume. All audio-driven visuals (LED grid, blocks, grid, giphy, sound investigations) read this value.
